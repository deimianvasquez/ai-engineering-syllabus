# Milestone 0 — Choose Your Company — Reference solution

This README describes what a correct reference submission should include for the **Milestone 0 — Choose Your Company** project.

## Expected implementation scope

No code is required in this milestone.

The student must create (and commit) a `company-choice.md` file in the root of their fork, based on the three company profiles available in `CONTEXT-*.md` within the template repository.

## What a correct `company-choice.md` must contain

### 1) A specific company selection

- A single company is selected (not a generic statement like “I will decide later”).

### 2) Clear justification (minimum 3 sentences)

- The written justification must be at least 3 sentences.
- It must reference specific details from the chosen company’s context file (not only the company name or sector).

### 3) Department discovery (minimum 2 departments)

- Identify at least two departments.
- For each department, add a brief explanation of why that department’s problems are interesting to you.

### 4) Milestone / automation challenge identification

- Name at least one specific milestone or automation challenge from the milestone map in the context document.

### 5) AI Agent proposal (inside `## My AI Agent Idea`)

- Add a section titled `## My AI Agent Idea`.
- Write 2–4 sentences explaining:
  - what the agent would do,
  - what information it would need access to,
  - what it would output or trigger,
  - how it would interact with users or other systems (in plain language).

## Validation checklist (reviewer-ready)

- [ ] `company-choice.md` exists in the root of the submitted fork.
- [ ] A specific company has been selected.
- [ ] The justification is at least 3 sentences and references the context file with concrete details.
- [ ] Two or more departments are identified with short explanations.
- [ ] At least one specific milestone / automation challenge is named.
- [ ] The submission includes a `## My AI Agent Idea` section with a 2–4 sentence agent proposal.
- [ ] Codespaces environment opens correctly with no errors (per project instructions).

## Notes for reviewers

- “Correct” here means _evidence of thorough reading_ of the relevant `CONTEXT-*.md` document.
- Generic reasoning that could apply to any company should be treated as incomplete.
