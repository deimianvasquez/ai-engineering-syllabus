# Generate Project Social Sharing

## Purpose

Automates project social publishing setup for `content/projects`:

1. Reads each `learn.json`
2. Uses English title (`title.en`) as cover title
3. Selects an image from `assets/cover/images` based on project metadata
4. Generates `preview.png` in the project `.learn` folder
5. Updates `learn.json` preview reference
6. Generates project-specific bilingual `sharing` messages in `learn.json`

This skill is aligned with LearnPack docs (`preview`, multilingual metadata, and `learn.json` conventions) and with the 4Geeks/BreatheCode workflow where social share copy can be carried in project metadata.

## Output contract (important)

- Write sharing texts only inside `learn.json` under the configured `sharing` key.
- Generate `preview.png` in the project `.learn` folder.
- Update `learn.json` with `preview` reference and `sharing` messages.
- If the user asks for social copy generation, the final source of truth must be the `learn.json` field, not a separate file.

## Run

From repository root:

```bash
python3 .cursor/skills/generate-project-social-sharing/scripts/generate_project_social_assets.py
```

Optional flags:

```bash
python3 .cursor/skills/generate-project-social-sharing/scripts/generate_project_social_assets.py \
  --project-root . \
  --author "by 4Geeks Academy" \
  --preview-filename "preview.png" \
  --preview-key "preview" \
  --sharing-key "sharing" \
  --sharing-brand "@4GeeksAcademy" \
  --sharing-link-template "https://github.com/4GeeksAcademy/{slug}" \
  --absolute-base-url "https://raw.githubusercontent.com/4GeeksAcademy/ai-engineering-syllabus/main" \
  --write-both-preview-keys
```

## Defaults and assumptions

- Cover template path: `assets/cover/cover-template.html`
- Images path: `assets/cover/images`
- Projects path: `content/projects`
- Output image path per project: `.learn/preview.png`
- `learn.json` key updated by default: `preview`
- `preview` is always written as absolute URL based on `--absolute-base-url`
- With `--write-both-preview-keys`, it updates both `preview` and `preview_url`
- `sharing` is generated by default as structured bilingual entries (`en`, `es`) inside `learn.json`
- Use `--skip-sharing` when only preview regeneration is needed
- English title fallback supports `title.us` when `title.en` is missing

## Image selection heuristic

The script picks an image using `technologies`, `slug`, `title.en`, and `description.en`:

- `command-line.svg` for terminal/CLI/bash topics
- `workflow.svg` for process/modeling/diagram topics
- `ai-communication.svg` for chat/prompt/API communication topics
- `ai-web-development.svg` for web/frontend HTML/CSS/Tailwind topics
- `ai-coding.svg` for coding/programming implementation topics
- `working-with-ai.svg` fallback

## Social sharing generation

The script writes `sharing` directly in `learn.json` as an array of bilingual message entries:

- Uses project title (`title.en`, `title.es` fallbacks)
- Includes project link from `--sharing-link-template` (`{slug}` placeholder)
- Includes project-aware hashtags from `technologies`
- Produces reusable social copy for student/project promotion

Example output shape:

```json
"sharing": [
  {
    "en": "I just completed ...",
    "es": "Acabo de completar ..."
  },
  {
    "en": "Built and shipped ...",
    "es": "Construido y publicado ..."
  }
]
```

## Validation checklist

- Confirm `npx playwright` is available
- Confirm `assets/cover/cover-template.html` exists
- Confirm images exist in `assets/cover/images`
- Verify at least one generated image and updated `preview` URL
- Verify `sharing` is present in `learn.json`
- Verify no standalone markdown file was created for sharing messages
